# Changelog

记录女专办 Android 客户端的版本变更。版本号遵循 [SemVer](https://semver.org/lang/zh-CN/)，`versionCode` 编码规则为 `major*1000 + minor*10 + patch`（例：1.3.0 → 1030）。**边界：patch ≤ 9、minor ≤ 99**——公式只给 patch 留了个位，越界会进位撞号（如 1.2.10 与 1.3.0 同为 1030），Android 会拒绝 versionCode 不增的“升级”；逼近上限时直接升 minor / major，或先改用更宽的编码。

## [1.11.0] - 2026-07-20

阅读体验批次：通知详情排版全面升级，校历不再外跳浏览器。

### 通知
- **正文字号可调**：详情页顶栏新增「调整字号」，小 / 标准 / 大 / 特大四档即点即变、边看边调；选择持久记住（外观偏好，登出不清），并叠加在系统字体大小之上，标题 / 段落 / 表格 / 附件名整篇同步缩放。
- **中文排版优化**：正文行高由 1.5 提至 1.7 倍字号，阅读更松弛；长链接 / 长英文词断行更自然；标题加粗强化层级。
- **还原公文对齐**：解析 Word 粘贴正文的 text-align / align / `<center>`（沿容器继承），通知里居中的小标题、右对齐的「教务处 + 日期」落款不再被压成左对齐。
- **表格更好读**：竖直分隔线跟随行高，多行单元格不再「断线」；超长单元格限宽换行，不再把整行拖成横滚长龙；表头行加底色与数据行区分。

### 校历
- **校历在 app 内直接看**：PDF 用系统 PdfRenderer 原生渲染（不引第三方库、不走 WebView），双指捏合 1–4 倍、双击快速放大，双向平移带惯性；文件落本地缓存，重进秒开、离线可看，下载字节带 PDF 魔数校验防脏缓存。早年 JPG 版校历走全屏图片查看器；DOC / XLS 等无法原生渲染的格式仍外跳系统应用，查看页顶栏保留「在浏览器中打开」逃生门。

## [1.10.0] - 2026-07-19

品牌升级批次：品牌标重绘、全套品牌资产矢量化、桌面图标→启动页→登录页视觉打通。规范落在新文档 [docs/brand.md](docs/brand.md)。

### 品牌
- **品牌标重绘「开卷有成」**：翻开的双层书页 + 上扬对勾（学业之上，事项办结），纯描边几何，去掉了旧标里小尺寸下发糊的手机边框元素。
- **桌面图标焕新**：从「白底红标」翻转为「女专红渐变底 + 白标」——桌面图标、启动页、登录页头部自此是同一块红底白标，从点开图标到进入应用视觉连续；辨识度也高于原先的白底。

- **启动页深浅色统一**：深色模式此前引用的是低分辨率的 launcher 单色位图，与浅色模式不同源；现在两种模式共用同一矢量启动图标。
- **修复登录页深色模式下品牌标近乎不可见**：旧实现白标叠在深色主题的浅粉 primary 上；现按 onPrimary 着色，两种模式对比度都正确。
- 「关于」弹窗顶部加入品牌标。
- **README 品牌化**：新增居中头图与版本 / CI / 平台徽章；新增 `docs/brand.md` 品牌规范（母版几何、色板、场景参数、改动同步清单）与 `docs/assets/brand/` 矢量资产。

## [1.9.1] - 2026-07-19

全项目体验审查产出的修复批次：失败路径不再静默、导航与配色多处修正。

### 全局
- **下拉刷新失败不再无声**：全 app 带刷新的页面（课表 / 成绩 / 考试出分 / 通知 / 考试 / 校历 / 培养方案 / 基本信息）刷新失败时弹出提示，旧数据保持展示——转圈停了不再被误读成「已是最新」（出分季反复刷「考试出分」时尤其重要）。
- **错误提示不再冒英文**：网络异常兜底文案统一为中文，不再出现 "Failed to connect to …" 之类的英文系统串直接上屏。
- **修复底部导航状态丢失**：首次安装（从登录页进入）的会话里，切换底栏 tab 会丢失各页滚动位置、返回键需倒退多次的问题。

### 课表
- **切换学期失败不再顶掉课表**：弱网下切学期失败时保留正在查看的课表并弹提示（旧版整页变错误页，且「重试」回到的还是旧学期）；切换期间顶栏有加载指示，忙碌中再点会提示「正在刷新」而不是没反应。
- **修复跨零点后「今」列错位**：页面驻留跨过周日零点后，「今」列与「本周」标记自动推进到新的一周（旧版会停留在上一周的网格上）；离线兜底态同样按快照开学日正确推进。
- 编辑周次新增「单周 / 双周」快捷预设。

### 成绩
- **等级制挂科恢复警示色**：「不及格 / 不合格 / 不通过」不再显示为与「良好」相同的颜色。
- 「考试出分」总评徽章按分数语义变色：挂科总评红底警示，不再与 95 分视觉相同。

### 考试
- 「已结束」考试改为最近考完的排最前（旧版按时间正序，昨天刚考的沉在学期初的记录底下）。

### 通知 / 图片
- 教务网图片解码加入降采样（最长边压到 2048px 内）：通知里的手机长截图不再挤爆图片内存缓存，滚动更顺滑，超大图不再因超出 GPU 纹理上限显示空白。

### 登录与我的
- 登录页密码框新增明文 / 密文切换（连续输错会触发统一身份认证风控，自查输入比盲试重要）。
- 退出登录弹窗如实列出将被清除的本地数据（手动编辑的课程周次、师生查询历史等教务系统不保存、不可恢复项）。
- 「我的」页考试卡片区分「暂无待考」与拉取失败：网络失败时显示中性的「考试安排」，不再误报「暂无待考」。

### 内部
- 移除未使用的 material3-adaptive-navigation-suite 依赖。

自动登录/会话保持整体重做：修复「实际使用中经常弹出重新登录」。教务系统会话约 20~30 分钟空闲即失效，旧版静默重登在真实网络环境下大量失败并把用户踢到登录页，本版从根上重做了恢复链路。

### 静默重登（核心修复）
- **免密 SSO 续票**：会话过期后优先用 CAS 长期票据（TGC）免密续票，不再每次都提交完整账号密码——更快（1 次请求 vs 5 次）、更稳，也不再刺激统一身份认证的登录风控。TGC 失效才退回完整账密登录。
- **并发合并**：课表 / 成绩 / 通知 / 图片同时会话过期时只做一次真实重登，其余请求复验后直接复用（旧版会背靠背连打 N 次完整 CAS 登录，易触发风控要求验证码）。
- **瞬时失败不再踢人**：网络抖动 / 教务网 5xx / 重登被节流时，页面显示可下拉重试的网络错误，**不再**清空本地会话并弹回登录页；网络恢复后下次请求自动续登。只有「确认密码已失效」才会要求重新登录。
- **凭证不再被误清**：旧版把「落回 CAS 登录页」一律当密码错误并删除已保存的密码（验证码要求 / 系统繁忙也中招），导致自动登录永久失效。现改为仅在确认凭证被拒时清除；验证码 / 风控 / 繁忙类拒绝保留密码等待自愈。
- **误踢守卫**：弱网下一个请求重登失败不再连带销毁另一个请求刚刚续上的新会话（踢人前先复验会话真死了没有）。
- 图片（头像 / 通知配图 / 师生照片）加载失败只显示占位，不再把用户从正在看的页面踢到登录页；会话半死时图片端点返回的登录页 HTML 也能正确触发重登（旧版表现为图片永远加载失败）。

### 冷启动与弱网
- **离线也能进 app**：有已保存密码但暂时没网（地铁 / 电梯 / 校园网切换）时直接进主界面，课表显示上次缓存（离线标记），网络恢复自动续登刷新——不再被挡在登录页手输密码。
- 冷启动会话有效时少发一次网络请求，进入更快。

### 稳定性
- 修复一个跨锁死锁：会话失效时多个页面同时刷新、且恰好需要重新登录时，可能整个应用卡住转圈只能杀进程（各 Repository 的缓存清理恢复为无锁，与自动重登不再互相等锁）。

### 登录页
- 新增**「使用已保存的密码登录」**一键重登按钮：真被登出时，本机存有加密密码即可一键回到 app，无需重新手输统一身份认证密码。

- 启动器图标重做：自适应图标改回多分辨率位图前景 + 纯白背景 + monochrome 单色版，启动屏复用同一资源；与 1.3.0 的矢量徽章方案相比小尺寸下细节更清晰。
- HTTP 链接出站和 302 重定向 `Location` 头都会把 `http://*.jxnu.edu.cn` 升级到 https；debug 日志里 `UserNum=` 学号脱敏。

### 修复
- 课表晚间段：段标签格 `<td>晚上</td>` 与节次标签格 `<td>晚<br>上</td>` 不再被混淆，修复周一晚课跑到周二并伴随幽灵课程「晚 / 上」的 Bug。
- 课表网格不再依赖 HTML4 `bgcolor` 属性区分有课/空格，改用语义判定（`<div>` + 课程名/教室/班级），对教务系统改主题更鲁棒。
- 课程详情、学期选择、周次编辑这几个 Bottom Sheet 打开时会先消化系统返回键关闭自己，不再直接退出 App。
- 周日历表头与 baselineWeek 对齐到同一套坐标（`previousOrSame(MONDAY)`），周一是开学之前那天时也不再错位。
- 考试列表条目 id 改用未清洗的原始时间字符串拼接，避免 `&nbsp;` 让同一行在不同次刷新里产生不同 id。

### 内部
- 网络层重构：抽出 `JwcClient` 门面统一业务请求；`SessionRecovery` 用 Mutex 合并并发重登；`JwcResponseGuard` 不再直接广播 expired，把信号交给 retry 流程；URL 常量集中到 `JxnuUrls`。
- `PersistentCookieJar` 改用 `ReentrantReadWriteLock`：CAS 重定向链上的多个并发 OkHttp 调用走读路径，过期 cookie 批量升级写锁清理。
- 课表晚间段新增 `NIGHT_FIXTURE` 回归测试，固化段标签 vs 节次标签的区分语义。
- `WidgetUpdateScheduler` 使用 `setAndAllowWhileIdle` 穿透 Doze 维护窗口，仍是 inexact 不需要 SCHEDULE_EXACT_ALARM 权限。
- 发布前 `test + compileDebugKotlin + lintDebug` 通过。
