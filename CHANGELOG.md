# Changelog

记录女专办 Android 客户端的版本变更。版本号遵循 [SemVer](https://semver.org/lang/zh-CN/)，`versionCode` 编码规则为 `major*1000 + minor*10 + patch`（例：1.3.0 → 1030）。

## [1.2.4] - 2026-07-07

### 登录与会话
- 修复一次会话过期后，旋转屏幕 / 深浅色切换 / 系统回收恢复等界面重建会把刚重新登录的有效会话再次注销的问题（陈旧过期信号在重建时被重复消费）——这是「间歇性掉登录」的主要来源。
- 修复会话半失效时成绩页被解析成「空成绩单」并写入缓存的问题：现在识别为页面异常并提示下拉刷新 / 重新登录，不再无提示地显示空数据。
- 会话过期不再当作退出登录处理：课程周次编辑、通知已读状态、桌面 widget 快照在同一账号重新登录后全部保留（换账号登录时仍会自动清空上一账号的本地数据）。

### 内部
- 错误提示优先展示业务层定制文案（如成绩壳页的刷新 / 重登引导），不再一律回落到通用解析失败文案。
- `AuthStorage.clear()` 不再连带清除 installation UUID 与换号检测标记。

## [1.2.3] - 2026-06-29

### 登录与会话
- 修复部分设备上「下次自动登录」无法长期生效的问题：当 Android Keystore / EncryptedSharedPreferences 因备份迁移、旧 keyset 损坏或系统密钥失效而无法打开时，现在会清理本地损坏的加密凭据并重建安全存储。
- 自动登录仍只使用 Android Keystore + EncryptedSharedPreferences；不会退化为明文保存密码。已损坏的旧凭据不可恢复，用户重新登录一次后即可重新保存。

### 内部
- 加密凭据存储新增自愈路径：先重建损坏的 `jxnu_secure_creds` 文件，必要时重置本 App 使用的默认 MasterKey alias，避免安全存储永久不可用。
- 保持备份排除规则不变，登录 cookie、自动登录凭据和 widget 快照仍不会随 Android cloud backup / device transfer 迁移。

## [1.2.2] - 2026-06-09

### 登录与隐私
- 修复退出登录可能被页面导航取消的问题：现在会等待 cookie、凭据、缓存、通知锚点和 widget 快照清理完成后再回到登录页。
- 退出登录或会话过期后会立即清空并刷新桌面课表 widget，避免桌面继续显示上一位用户的今日课表。
- 阻止退出登录后仍在后台排队的课表快照写回，避免旧课表重新落盘。

### 师生查询
- 修复师生详情、课表和头像 URL 中 `UserNum` base64 参数未编码的问题，避免包含 `+`、`/`、`=` 的结果打不开。

### 内部
- 通知详情缓存清理改为与读写共用同一把锁，减少退出登录时旧详情回写缓存的竞态。
- `UserNum` URL 构造新增回归测试，头像 URL base64 编码改为 JVM/Android 都可测试的实现。

### 个人中心

## [1.0.0] - 2026-05-23

首次公开发布。聚合此前 1.x 系列的所有能力作为正式 1.0；以下按用户可感知的功能模块组织。1.x 历史条目保留在下方作为内部开发日志。

### 课表
- 周视图 + 12 节制时间线，高亮「上课中 / 下一节」。
- 学期下拉切换、「今天」一键回到当前学期 + 当前周；周次芯片自动滚动至选中周。
- 课程详情可编辑「上课周次」并本地持久化（教务系统不返回真实周次）。
- 顶部横幅：7 天内有考试时显示倒计时，点击直达考试列表。
- 周次表头按学期映射到现实日历日期（`M/d`）。

### 通知
- 教务通知 + 通告 + 图文新闻 3 个来源合并按日期降序展示，支持类型过滤芯片。
- 首屏先返回纯文本通知，再补充图文新闻；红点提示未读。
- 详情页 WebView，禁用 JavaScript / 文件访问 / 混合内容。

### 成绩与培养方案
- 学期成绩 + 考试出分（test_grades）双视图。
- 顶部统计卡：已修学分、加权平均标准分、毕业最低学分与完成率。
- 培养方案查看：毕业要求、课程模块、课程清单、学位课程与完成状态。

### 考试
- 学期考试安排 + 补缓考分区合并。
- 按日期划分「未开始 / 今天考 / 已结束」状态（教务系统返回的时间不可信，仅按日期）。

### 个人中心与查询
- 师生查询入口（教工 / 学生 · 模糊与精确），结果可直接打开他人课表。
- 校历入口、空闲教室入口（WebView 接入社区项目 [xmy3/jxnu-classroom](https://github.com/xmy3/jxnu-classroom) 的 GitHub Pages 数据）。
- 学生头像默认不加载，需在「显示学生头像」开关打开；学院 / 专业 / 班级由成绩页后台异步补全。

### 桌面 widget
- 今日课表 + 下节课倒计时 Glance widget。
- AlarmManager 自调度机制：在每个节次开始 / 结束 / 跨日时点自动刷新，不依赖精确闹钟权限。

### 登录与安全
- JXNU CAS SSO 登录（RSA 加密密码 + 设备 visitor id），无验证码。
- 可选「下次自动登录」：账号密码通过 Android Keystore + EncryptedSharedPreferences 加密保存；加密存储不可用时自动禁用。
- 会话过期时静默重登并重放原请求；网络层强制 HTTPS、拒绝用户根证书；release 包关闭业务日志、debug 日志中 `UserNum=` 学号脱敏。
- 退出登录会清空所有派生缓存、cookie、本地周次覆盖、通知红点锚点与 widget 快照。

### 主题
- 深色 / 浅色 / 跟随系统三档；Material You 动态取色（Android 12+）。

### 隐私
- 应用没有自有后端，所有数据直接来自学校教务系统并仅保存在本机。
- 不上传账号、密码、cookie、学号、成绩或课表到任何第三方。
- 加密凭据 / cookie / widget 快照 / 主题与头像偏好 / 通知已读锚点 / 课表周次覆盖全部排除在 Android 云备份与设备迁移之外。

### 免责声明
- 本应用与江西师范大学及其教务处无任何隶属或合作关系，仅供个人学习查询使用。

## [1.3.3] - 2026-05-22

### 新增
- 课表顶部新增「即将考试」横幅：7 天内有考试时显示倒计时，点击直达考试列表。
- 课表标题支持点击切换学期：下拉所有可选学期，已访问过的学期秒切，本学期带「本学期」标记。
- 课表周次表头按当前学期显示每天具体日期（`M/d`），跨周次浏览也能对上现实日历。
- 课程详情新增「编辑周次」入口，自定义周次会本地持久化；编辑器内置「全选 / 1-8 / 慕课」快捷预设。
- 登录页新增「使用帮助」对话框，覆盖账号密码、登录失败排查、隐私数据三段说明。

### 改进
- 会话过期自动恢复：业务请求遇到 SessionExpired 时若本地有保存凭据，会静默重登并重放原请求，用户感知不到中间断点；并发请求被同一把 Mutex 合并为一次重登。
- 课表周次芯片自动滚动至选中周居中，且能露出左右各两周的上下文。
- 高亮标签「上课中 / 下节」改为在课程卡顶部独占一行，不再覆盖课程名。
- 学期 / 周次切换以及下拉刷新不再闪 Loading，旧数据保留直至新数据就绪。
- 退出登录现在会清掉所有派生数据：6 个仓库内存缓存、课程周次本地覆盖、通知已读锚点、桌面 widget 快照。
- 启动器图标重做：自适应图标改回多分辨率位图前景 + 纯白背景 + monochrome 单色版，启动屏复用同一资源；与 1.3.0 的矢量徽章方案相比小尺寸下细节更清晰。
- HTTP 链接出站和 302 重定向 `Location` 头都会把 `http://*.jxnu.edu.cn` 升级到 https；debug 日志里 `UserNum=` 学号脱敏。

### 修复
- 课表晚间段：段标签格 `<td>晚上</td>` 与节次标签格 `<td>晚<br>上</td>` 不再被混淆，修复周一晚课跑到周二并伴随幽灵课程「晚 / 上」的 Bug。
- 课表网格不再依赖 HTML4 `bgcolor` 属性区分有课/空格，改用语义判定（`<div>` + 课程名/教室/班级），对教务系统改主题更鲁棒。
- 课程详情、学期选择、周次编辑这几个 Bottom Sheet 打开时会先消化系统返回键关闭自己，不再直接退出 App。
- 周日历表头与 baselineWeek 对齐到同一套坐标（`previousOrSame(MONDAY)`），周一是开学之前那天时也不再错位。
- 考试列表条目 id 改用未清洗的原始时间字符串拼接，避免 `&nbsp;` 让同一行在不同次刷新里产生不同 id。

### 内部
- 网络层重构：抽出 `JwcClient` 门面统一业务请求；`SessionRecovery` 用 Mutex 合并并发重登；`JwcResponseGuard` 不再直接广播 expired，把信号交给 retry 流程；URL 常量集中到 `JxnuUrls`。
- `PersistentCookieJar` 改用 `ReentrantReadWriteLock`：CAS 重定向链上的多个并发 OkHttp 调用走读路径，过期 cookie 批量升级写锁清理。
- 课表晚间段新增 `NIGHT_FIXTURE` 回归测试，固化段标签 vs 节次标签的区分语义。
- `WidgetUpdateScheduler` 使用 `setAndAllowWhileIdle` 穿透 Doze 维护窗口，仍是 inexact 不需要 SCHEDULE_EXACT_ALARM 权限。
- 发布前 `test + compileDebugKotlin + lintDebug` 通过。
